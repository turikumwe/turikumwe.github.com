Getting started with GitHub

Introducing Github

GitHub is a web-based hosting service for software development projects that use the Git revision control system, for any one who is working on a software project, i am encouraging you to use this version control system because it will help you to maintain your codes. It is the most popular, secure and easy to use. Let’s follow the instruction tu set up your first github repository and start using it.

Activities:

1. Creating your github account
2. Setting up Git
3. Creating your repository on www.github.com
4. uploading your files.

1. Creating your github account

1.1 Open your browser and logon https://github.com/
1.2 choose Signup and choose create a free account.
1.3 Provide all the required information and click create an account
1.3 You are done with the account creation.

2. Setting up Git

Setting Git on your system, involves the following steps:

2.1. Download and Install Git
At the heart of GitHub is an open source version control system (VCS) called Git. Created by the same dudes that created Linux, Git is responsible for everything GitHub related that happens locally on your computer. you can read more about Git on Git details
2.1 Download and Install The latest version from here here

Next: Set up your SSH Keys

We use SSH keys to establish a secure connection between your computer and GitHub. Setting them up is fairly easy, but does involve a number of steps.

2.2 Open your terminal
to check whether there are already an SSH directory you will need to use this codebox:

$ cd ~/.ssh
(Assume that you are in your home directory, if not so use $ cd ~/ to go in your home directory)
if it says“No such file or directory“ you will continue, otherwise follow this instruction to backup the existing ssh keys.
To backup:
$ ls

Config id_rsa id_rsa.pub known_host
$ cp id_rsa* key_backup
$ rm id_rsa*

3.3 Generate your new SSH Keys

To generate a new SSH key, enter the code below. We want the default settings so when asked to enter a file in which to save the key, just press enter.
$ ssh-keygen -t rsa -C “your_email@yourdomain.com“

The output will look like this:
Generating public/private rsa key pair.
Enter file in which to save the key
(/User /your_user_directory/.ssh/id_rsa):

press enter to continue

You will be asked to put your passPhrase(It is like password but use a phrase to maximize the its security).
Enter passphrase twise or leave leave it empty(Think about the risk of having no password!)

2.4 Add your SSH key to gitHub.

On the GitHub site Click“Account Settings“ > “SSH Public Keys“ > “Add another public key“
Open the id_rsa.pub file with a text editor (Notepad, TextEdit, or gedit will do just fine). This is your public SSH key. You may need turn on “view hidden files” to find it because the .ssh directory is hidden. It’s important you copy your SSH key exactly as it is written without adding any newlines or whitespace.

or use the following codebox to unhide the hidden file:

$ defaults write com.apple.Finder AppleShowAllFiles TRUE

Relaunch the finder and you will able to view the file.


$ defaults write com.apple.Finder AppleShowAllFiles FALSE

Copy the whole its contents and paste in the provided field on GitHub website and provide any name, then click Add key.

Test every thing by typing this codebox:

$ ssh -T git@github.com

then gitHub will tell you your authentication status, if it is successful you are done with your SSH Keys.

3. Creating your repository on www.github.com

3.1 Login to your www.github.com account
3.2 Click New Repository.
3.3 for the project name you can type your username anc choose create Repository. Note that the description and the homepage URL are optional.
3.4 After creating this repository, you’ll be redirected to the page that will help you to get started

4. Creating and uploading your files.

Global setup:

On your terminal:

$ git config —global user.name “Your Name“
$ git config —global user.email srkh@yahoo.com

Global setup:


issue the following codebox to create directory which you will use with your GitHub files:

$ mkdir Your_direcory_name
$ cd Your_direcory_name
$ git init
$ touch README
$ git add README
$ git commit -m ‘first commit’
$ git remote add origin git@github.com:your_github_user_name/your_github_user_name.git
$ git push -u origin master

File to upload

4.1 in the created directory on (mkdir Your_direcory_name) step, create your first file called index.html and put some contents
Issue the following codebox to check available files:

$ ls
Issue the following codebox to check available files to commit: This will display the uncommitted files
$ git status
Issue the following codebox to add files to be committed:
$ git add index.html
Issue the following codebox to commit your files:
$ git commit -m “Type your comments“
The last step is to Issue the following codebox to push on gitHub server
$ git push

If those steps are successful, then you can browse your files using the following directory


your_username.gitHub.com
